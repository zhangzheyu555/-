# 微信小程序一期设计

状态：代码完成，未发布。交付目标仅为 `mp-weixin`；H5、Android、iOS 保留 UniApp 兼容层，不属于本期发布物。

## 架构边界

`mobile-uniapp` 是唯一移动业务前端，复用 Vue 3、TypeScript、Pinia、`api/client` 与 `platform/session`。它与 `frontend-vue` 通过同一 Spring Boot `/api/**`、同一 MySQL、同一附件服务和同一 `operation_log` 互通。客户端只持久化 Bearer Token；任何库存、单据、巡检、附件、学习进度、成绩或摘要都只从服务端读取。

小程序主包只有登录、工作台、待办、应用、我的。固定底栏为：工作台、待办、应用、我的。`/api/auth/me` 返回的角色、权限和数据范围决定工作台及应用入口；页面可见性从不替代后端 401/403 校验。

| 分包 | 内容 | 预载策略 |
| --- | --- | --- |
| `pkg-store` | 门店库存、叫货、收货 | 按入口加载 |
| `pkg-warehouse` | 叫货发货、退货收货、预警待办 | 按入口加载 |
| `pkg-inspection` | 巡检、整改提交、整改复核 | 从应用页在 Wi-Fi 下预载 |
| `pkg-learning` | 视频、考试、员工助手 | 按入口加载 |
| `pkg-summary` | 财务和老板只读摘要 | 按入口加载 |

## 业务闭环

- 店长：库存/预警 → 叫货（稳定 `clientRequestId`）→ 服务端发货 → 确认收货；库存和日志仅由服务端事务更新。
- 仓库管理员：查看授权范围内的叫货单，确认发货；处理授权范围内的退货收货。不开放商品、仓库、采购基础配置。
- 督导/运营：巡检照片先上传正式附件服务，再由 AI 提示、人工确认后保存；运营复核整改通过或驳回。
- 店长整改：仅访问本店待整改记录；照片经专用整改附件接口上传，以附件 ID 提交。状态为待整改、待运营复核、整改已通过、整改已驳回，且不改历史得分、扣分、标准版本或原始结论。
- 员工：视频使用短时播放票据，主 Token 不进 URL；学习进度、交卷和成绩均以 MySQL 为准。
- 财务/老板：只读摘要及待办，不展示经营录入、导入、权限、配置或批量导出。

## 交互与安全

页面采用安全区和 8px 间距体系；关键点击目标最小 44px，具备加载骨架/状态提示、失败重试与禁用态。图片仅上传附件，不传 Base64、文件名或临时路径作为业务关联。弱网时叫货与巡检保留稳定业务键；考试、收货和后端状态转换复用服务端幂等/行锁。

候选包使用 Node 20、锁文件和临时 Git 源码目录构建。`MOBILE_STAGING_MP_WEIXIN_APP_ID` 与 `MOBILE_STAGING_MP_WEIXIN_API_BASE_URL` 仅从进程环境或受控 CI 注入临时目录；构建脚本拒绝空值、`localhost`、局域网、`.invalid`、携带凭据或非 HTTPS 地址，不在源码、日志或候选清单中写入 AppID 或完整地址。

当前没有 QA HTTPS 域名、AppID/合法域名和真机证据，因此状态为：**不可上传体验版**。
