# 旧版迁移清单

## 分类

| 位置 | 当前定位 | 后续处理 |
| --- | --- | --- |
| `frontend-vue/` | 正式 Vue3 前端 | 继续开发、测试和发布 |
| `backend/` | 正式后端 | 继续开发、测试和发布 |
| 顶层 `index.html` | 最早旧版入口 | 仅保留视觉和交互参考 |
| 顶层 `database.js` | 旧版基础数据与浏览器存储逻辑 | 仅用于梳理历史字段，禁止新代码运行时读取 |
| `runtime-static/` | 外置旧静态页副本 | 冻结，后续移动到只读 `/legacy` 发布目录 |
| `backend/src/main/resources/static/` | jar 内旧静态页副本 | 冻结，后续只提供 `/legacy` 回退 |
| `web/` | 实验 Vue 前端 | 停止开发，不作为生产入口 |

## 可迁移内容

- 颜色、字体、密度、表格、表单排列和业务操作节奏。
- 巡店、工资、报销、门店详情等旧版已有的真实业务字段。
- 已验证的业务规则，但必须改为后端接口和 MySQL 持久化实现。

## 禁止迁移内容

- `database.js` 中的 localStorage、模拟数据、固定账号和默认密码。
- 旧页的全局数据对象、正则读取标准、前端利润计算和跨门店无权限访问。
- 平台接口失败后的假营业数据、假成功状态或假保存按钮。

## 迁移完成判定

一个功能只有同时满足以下条件才视为完成迁移：Vue3 有可用页面、后端有受权限保护的真实接口、MySQL 有稳定结构、关键操作进入审计日志、刷新页面数据不丢失，并有测试覆盖越权与失败路径。
