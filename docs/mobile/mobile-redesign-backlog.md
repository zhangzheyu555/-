# 手机端适配重构任务清单

> **状态：已延期**  
> 本轮（2026-07-10）不进行多分辨率和手机端验收。  
> 以下任务统一延后至专门的移动端优化迭代中处理。

---

## 背景

当前系统（Vue 3 + TypeScript + Pinia）以桌面端为主要使用场景开发。  
手机端适配需要系统性重构，而非临时加媒体查询。  
本轮已明确不做以下任何手机端相关工作。

---

## 待办任务清单

### 1. 移动端导航

- 底部 TabBar 导航替换左侧 Sidebar
- 面包屑导航的移动端简化
- 返回按钮与页面栈管理
- 手势返回支持

### 2. 页面信息层级

- 卡片布局优先级重排（桌面 4 列 → 手机 1 列）
- 关键指标前置（首屏可见核心数据）
- 次要信息折叠/收起
- 长文本截断与展开

### 3. 表格转换

- 宽表格 → 卡片列表（Card List）转换
- 横向滚动 vs 列隐藏策略
- 固定首列（门店名/商品名）方案
- 表头吸顶（Sticky Header）

### 4. 表单操作

- 表单单列布局（标签在上，输入框在下）
- 日期选择器移动端适配
- 下拉选择器移动端原生体验
- 文件上传移动端适配（拍照/相册）

### 5. 底部安全区域

- `env(safe-area-inset-bottom)` 适配
- iPhone X+ 底部横条避让
- 固定底部按钮的安全间距
- Android 导航栏避让

### 6. 触摸交互

- 点击区域最小 44×44px（iOS HIG）
- 长按菜单（操作项）
- 滑动操作（左滑删除/右滑操作）
- 下拉刷新
- 触感反馈（Haptic Feedback）

### 7. 响应式断点

- 统一断点系统设计（建议：480 / 768 / 1024 / 1280）
- 替代当前分散的 `@media` 查询
- CSS 变量驱动的响应式间距系统
- 容器查询（Container Queries）探索

### 8. 不同手机尺寸

- iPhone SE (375×667)
- iPhone 12/13/14 (390×844)
- iPhone 14 Pro Max (430×932)
- Android 小屏 (360×640)
- Android 大屏 (412×915)
- 折叠屏适配策略

### 9. 双重滚动条

- 页面级滚动 + 区域内滚动的冲突解决
- `overscroll-behavior` 控制
- 固定高度的 flex 布局替代 `calc(100vh - xxx)`
- iOS 弹性滚动（`-webkit-overflow-scrolling`）

### 10. 移动端性能

- 虚拟列表（长列表优化）
- 图片懒加载
- 组件按需加载/代码分割
- 减少重排重绘
- 移动端网络优化（请求合并/缓存策略）

## 当前页面已知移动端问题

| 页面 | 问题描述 |
|------|---------|

| ProfitTablePage | 多列数据表在小屏无法阅读 |
| 所有表单页 | 双列布局在小屏拥挤 |

## 实施建议

1. **单独迭代**：分配 1-2 周专门做移动端适配，不与其他功能混合
2. **组件先行**：先建立移动端基础组件（TabBar、CardList、MobileForm）
3. **页面逐个迁移**：按使用频率排序（仪表盘 → 利润表 → 待办 → 仓库 → 设置）
4. **渐进增强**：桌面端功能不受影响，移动端作为增强层
5. **Playwright E2E**：增加移动端视口测试用例（iPhone 12 / Pixel 5）
