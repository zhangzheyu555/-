# 角色权限矩阵

## 基本原则

- 前端菜单用于减少干扰；后端认证和授权是唯一安全边界。
- 未登录返回 `401`；角色不匹配、跨门店、跨租户和附件越权返回 `403`。
- 店长只能访问 `user_store_scope` 中的门店；其他角色的数据范围由后端显式定义，不能依赖前端传参。
- 下表的“查看”不代表可以导出、审批或修改；这些动作均应单独检查并记录操作日志。

| 能力 | 系统管理员 | 老板 | 财务 | 店长 | 督导 | 仓库管理员 | 运营 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 全部门店经营总览 | 是 | 是 | 是 | 否 | 否 | 否 | 否 |
| 本店经营录入 | 配置 | 是 | 是 | 仅所属门店 | 否 | 否 | 否 |
| 利润表与明细 | 配置 | 是 | 是 | 仅所属门店 | 否 | 否 | 否 |
| 报销申请 | 否 | 是 | 是 | 仅所属门店 | 否 | 否 | 否 |
| 报销审批 | 否 | 是 | 是 | 否 | 否 | 否 | 否 |
| 工资与数据核对 | 否 | 是 | 是 | 仅本人授权范围查看 | 否 | 否 | 否 |
| 巡检、整改、复核 | 否 | 查看 | 否 | 查看本店整改 | 是 | 否 | 否 |
| 商品、采购、库存、出入库 | 配置 | 查看 | 查看单据 | 本店叫货和收货 | 否 | 是 | 否 |
| 培训、考试、盘存 | 配置 | 查看 | 否 | 本店任务 | 任务相关 | 查看 | 是 |
| 门店、员工、账号、角色 | 是 | 查看 | 否 | 否 | 否 | 否 | 部分基础配置 |
| 平台配置与数据辅助 | 是 | 查看 | 否 | 否 | 否 | 否 | 是 |
| 操作日志 | 是 | 是 | 财务相关 | 本人/本店可见范围 | 巡检相关 | 仓库相关 | 运营相关 |

## 菜单分组

| 分组 | 菜单 | 角色 |
| --- | --- | --- |
| 工作台 | 今日待办、风险提醒、待复核 | 按角色分发 |
| 经营中心 | 利润概览、利润表、门店详情、数据录入 | 老板、财务、店长 |
| 财务中心 | 报销、工资、数据核对、数据导出 | 老板、财务；店长仅报销 |
| 执行中心 | 督导巡店、仓库、培训、考试、盘存 | 督导、仓库、运营、店长按任务 |
| 管理中心 | 门店、员工、账号权限、平台配置、操作日志 | 系统管理员、老板、运营按权限 |

## 实施约定

后端使用统一认证上下文和权限服务执行角色与门店范围检查。新增接口必须先定义动作权限、门店范围和审计事件，再实现 Controller。
