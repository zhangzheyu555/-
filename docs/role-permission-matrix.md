# 角色权限矩阵

## 基本原则

- 前端菜单用于减少干扰；后端认证和授权是唯一安全边界。
- 未登录返回 `401`；角色不匹配、跨门店、跨租户和附件越权返回 `403`。
- 店长只能访问 `user_store_scope` 中的门店；督导只能访问明确授予的门店、品牌和仓库范围，不能依赖前端传参。
- 正式角色仅为 `BOSS`、`FINANCE`、`WAREHOUSE`、`STORE_MANAGER`、`SUPERVISOR`、`EMPLOYEE`。`ADMIN`、`OWNER`、`OPS`、`OPERATIONS` 仅用于历史兼容归一，不能创建、编辑或授权。
- 下表的“查看”不代表可以导出、审批或修改；这些动作均应单独检查并记录操作日志。

| 能力 | 老板 | 财务 | 店长 | 督导 | 仓库管理员 | 员工 |
| --- | --- | --- | --- | --- | --- | --- |
| 全部门店经营总览 | 是 | 是 | 否 | 授权范围辅助 | 否 | 否 |
| 本店经营录入 | 是 | 是 | 仅所属门店只读 | 否 | 否 | 否 |
| 报销、工资与数据核对 | 是 | 是 | 报销申请/本店查看 | 否 | 否 | 否 |
| 巡检、整改、复核 | 查看 | 否 | 查看本店整改 | 授权门店 | 否 | 否 |
| 日报损查询与月度 Excel 导出 | 是 | 是 | 否 | 否 | 否 | 否 |
| 商品、采购、库存、出入库 | 查看 | 查看单据 | 本店叫货和收货 | 盘存及授权范围查看，不得总仓管理 | 是 | 否 |
| 平台配置、配方、培训考试 | 查看 | 否 | 本店任务 | 授权范围管理 | 查看 | 仅本人学习 |
| 门店、员工、账号、角色 | 是 | 否 | 否 | 不得账号管理 | 否 | 仅本人资料 |
| 操作日志 | 是 | 财务相关 | 本人/本店可见范围 | 督导相关 | 仓库相关 | 本人相关 |

## 菜单分组

| 分组 | 菜单 | 角色 |
| --- | --- | --- |
| 工作台 | 今日待办、风险提醒、待复核 | 按角色分发 |
| 经营中心 | 利润概览、利润表、门店详情、数据录入 | 老板、财务、店长 |
| 财务中心 | 报销、工资、数据核对、数据导出 | 老板、财务；店长仅报销 |
| 执行中心 | 督导工作台、巡检、仓库、培训、考试、盘存 | 督导、仓库、店长按任务 |
| 管理中心 | 门店、员工、账号权限、平台配置、操作日志 | 老板；平台配置由督导按范围使用 |

## 实施约定

后端使用统一认证上下文和权限服务执行角色与门店范围检查。新增接口必须先定义动作权限、门店范围和审计事件，再实现 Controller。
