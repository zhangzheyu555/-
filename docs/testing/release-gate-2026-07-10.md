## “147 个红色测试”分类现状

当前仓库没有 Cloud 报告、测试名称、失败堆栈或截图索引，不能逐项判断 147 项属于真实缺陷、环境不兼容还是旧 HTML 测试。

仓库中唯一相关历史记录是 `docs/testing/static-resource-test-debt.md`：它记录旧快照中 210 项测试、26 个错误，根因是测试读取已废弃的 `static/index.html` 和 `static/database.js`。当前后端套件已变为 204 项且全部通过。这 26 项历史债务不能自动等同于 Cloud 的 147 项。

要完成逐项分类，必须提供 Cloud 导出的 JUnit XML、JSON/CSV 报告或至少包含测试全名和失败日志的文本文件。拿到报告后按以下规则处理：

- 真实缺陷：先复现，补回归测试，再最小修复。
- 环境不兼容：修正测试环境或增加明确前置条件，不标记为通过。
- 旧 HTML 废弃测试：确认对应 Vue3 覆盖后删除/迁移测试，禁止恢复旧 HTML。

## 继续执行所需的受控环境

继续 P0 前必须在启动测试进程的同一 PowerShell 会话中提供以下环境变量，值不得写入仓库或报告：

- `APP_ENV=TEST`，作为本轮人工门禁标识。
- `MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_DATABASE`、`MYSQL_USERNAME`、`MYSQL_PASSWORD`；数据库名必须明确包含 `test` 或 `qa`，且不得指向生产地址。
- 已暴露测试账号的用户名、当前密码和随机生成的新密码，分别通过临时环境变量提供；新密码不得复用任何默认值。
- 空库名称、升级库名称，以及脱敏旧库备份的本地路径。
- Cloud 失败测试的 JUnit XML、JSON、CSV 或完整日志路径。

修改前的 `application.yml` 在未设置 `MYSQL_*` 时会回退到本机默认库。由于 3306 上数据库用途未知，本轮没有使用当前 PowerShell 启动正式后端，避免误连和误写。前端开发服务已确认来自当前项目并恢复在 `http://127.0.0.1:5173`。

该默认回退已在源码中移除：`APP_ENV` 和五项 `MYSQL_*` 均为必填，自动创建数据库已关闭，TEST/QA 环境还会校验数据库名包含 `test` 或 `qa`。当前 8080 是修改前已由其他并发流程启动的旧构建，本轮没有终止或重启它，也没有通过它写入业务数据。
