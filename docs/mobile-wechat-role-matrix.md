# 微信小程序一期角色矩阵

| 角色 | 可用移动入口 | 允许动作 | 明确不开放 |
| --- | --- | --- | --- |
| 店长 | 库存、叫货收货、整改、待办 | 本店叫货、确认收货、上传并提交整改证据 | 经营录入、仓库配置、跨店整改 |
| 仓库管理员 | 仓库待办、待办 | 发货、退货收货、查看预警 | 商品/仓库基础配置、采购配置 |
| 督导/运营 | 巡检、整改复核、待办 | 巡检拍照、AI 建议人工确认、整改通过/驳回 | 代店长提交整改、跨授权门店操作 |
| 员工 | 学习、考试、员工助手 | 视频学习、交卷、查看成绩、提问 | 查看他人成绩、管理考试 |
| 财务 | 只读摘要、待办 | 查看权限范围内摘要和待办 | 经营录入、月度导入、批量导出 |
| 老板 | 只读摘要、待办 | 查看全范围摘要、培训摘要、待办 | 手机端账号权限、月度导入、批量导出 |

前端通过 `/api/auth/me` 的 `role + permissions + dataScopes` 隐藏无关入口；所有 Controller 仍从认证上下文读取用户。无 Token 为 401，越权/跨店/附件越权为 403，并由权限边界写拒绝日志。
