# Vue3 手机端 H5 工作流设计 v1

本阶段不做微信小程序，不做原生 App。手机端统一在 `frontend-vue` 中通过响应式布局、移动端菜单和角色路由实现。

## 目标屏宽

- 375px：常见小屏手机。
- 390px：主流 iPhone 宽度。
- 430px：大屏手机。
- 768px：平板和窄屏后台。

## 通用原则

- 手机端只展示当前角色高频动作，隐藏后台配置型菜单。
- 今日待办只提醒，不承载复杂业务表单。
- 业务处理回到对应中心：仓库中心、督导巡店、运营中心、门店详情。
- 所有真实数据写入 MySQL，前端不使用 localStorage 保存业务记录。
- 写入类动作必须有清晰的成功/失败提示，接口失败不能假装保存成功。

## 店长 H5

入口建议：

- 本店库存
- 向公司仓库叫货
- 我的叫货单
- 确认收货
- 门店盘存
- 门店自采
- 损耗上报
- 售后话术查询
- 培训考试

第一版流程：

1. 店长登录后默认进入仓库中心。
2. 本店库存只显示本门店库存，同时显示公司仓库可配送数量作为参考。
3. 叫货提交后进入“我的叫货单”，等待仓库处理。
4. 仓库发货后，店长在“确认收货”中核对商品和数量。
5. 确认收货后增加本店库存，并写门店库存流水。
6. 盘存、自采、损耗后续按独立 H5 表单补齐。

## 仓库 H5

入口建议：

- 仓库工作台
- 待处理叫货单
- 拣货/发货
- 采购入库
- 库存预警
- 出入库记录
- 配送退货单

第一版流程：

1. 查看待审核、待发货叫货单。
2. 审核数量，库存不足时提示缺货。
3. 发货时按批次先进先出扣减总仓库存。
4. 发货后生成出库流水和门店待收货任务。
5. 采购入库写入批次库存，库存预警自动刷新。

## 督导 H5

入口建议：

- 巡检任务
- 新建巡检
- 上传图片
- 违规记录
- 整改跟踪
- 复查

第一版流程：

1. 选择负责门店。
2. 填写巡检日期、评分、问题说明。
3. 上传图片或后续接 AI 识别。
4. 不合格问题生成整改记录。
5. 整改照片未上传时，巡检问题不能完结。

## 员工 H5

入口建议：

- 售后话术查询
- 培训学习
- 考试系统

第一版流程：

1. 员工通过简单入口查询售后问题。
2. AI 或规则库返回标准话术和处理建议。
3. 超过当前角色赔偿权限时，只能发起上级审批。
4. 培训学习记录和考试成绩写入 MySQL。

## 后续路由建议

- `/warehouse`：店长和仓库共用，根据角色显示不同工作台。
- `/inspection`：督导手机端巡检。
- `/operations?tab=training`：培训学习。
- `/operations?tab=exam`：考试。
- `/store-detail`：本店经营和库存摘要。
- `/mobile/after-sales`：后续新增售后话术 H5 页。
- `/mobile/loss`：后续新增损耗上报 H5 页。
- `/mobile/self-purchase`：后续新增门店自采 H5 页。

## 验收标准

- 375、390、430、768 宽度无整页横向溢出。
- 店长不显示老板后台、财务后台、商品维护、采购成本。
- 仓库不显示财务和老板驾驶舱。
- 督导不显示仓库采购、财务工资。
- 员工入口后续只显示培训、考试、售后。
- 所有写入动作走后端接口，不能把业务数据写入 localStorage。
