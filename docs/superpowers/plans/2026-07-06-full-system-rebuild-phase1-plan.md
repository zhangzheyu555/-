# 门店利润系统全面重构阶段 1 实施计划

日期：2026-07-06

## 阶段目标

阶段 1 只做新系统骨架，不迁移全部业务。完成后应同时满足：

- 旧 `index.html` 仍可打开和使用。
- 新 `web/` 前端可以独立启动，展示分组工作台。
- 新前端采用专业密集型后台风格。
- 后端开始按领域模块化单体整理。
- 后端提供新系统需要的基础 system/session 接口。
- 后续模块可以按经营财务、组织权限、运营人事逐步迁移。

## 本阶段范围

### 前端

新增 `web/`：

- Vue 3 + TypeScript + Vite。
- Pinia 管理会话和当前筛选。
- Vue Router 管理模块页面。
- AppShell 分组工作台布局。
- 基础页面：
  - 经营驾驶舱。
  - 利润录入。
  - 门店档案。
  - 数据助手。
- 基础组件：
  - 指标卡。
  - 筛选条。
  - 数据表。
  - 状态标签。
  - 空状态。

第一版使用真实业务字段和少量示例数据，先不直接替换旧前端主入口。

### 后端

在 `backend` 中新增领域模块骨架：

- `platform`：系统信息、会话模型。
- `organization`：品牌、门店模型边界。
- `finance`：利润、报表模型边界。
- `people`：工资模型边界。
- `operations`：报销、巡店、文件模型边界。
- `audit`：操作日志边界。
- `migration`：旧 JSON 迁移边界。

新增基础接口：

- `GET /api/system/overview`
- `GET /api/session/me`

这两个接口用于新前端骨架显示系统状态和模拟当前登录用户。正式认证在阶段 2 实现。

### 文档

- 更新 README，说明旧前端和新前端的启动方式。
- 新前端 README 说明开发命令。

## 本阶段不做

- 不删除旧 `index.html`。
- 不删除 `database.js`。
- 不把所有数据从 `kv_storage` 拆到业务表。
- 不实现正式 JWT 登录。
- 不完整迁移利润、工资、报销、巡店业务接口。
- 不把新前端设为唯一入口。

## 验收标准

### 后端

- `mvn test` 通过。
- `GET /api/health` 正常。
- `GET /api/system/overview` 正常。
- `GET /api/session/me` 正常。

### 新前端

- `npm install` 成功。
- `npm run build` 通过。
- `npm run dev` 能启动。
- 浏览器打开新前端后显示分组工作台。
- 侧边栏分组、路由切换、筛选控件、表格和数据助手输入框可交互。

### 旧前端

- 旧 `http://localhost:5500/index.html` 仍可打开。
- 旧前端不依赖新前端构建结果。
