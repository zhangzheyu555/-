## 当前验收锚点

- `/profit` 必须包含“合并营业收入”。
- `/profit-table` 必须包含“利润表”，不能显示财务工作台标题。
- `/store-detail` 必须包含“累计营收”。
- `/data-entry` 必须包含“实时利润核算”。
- `/expenses` 必须包含“新增报销”。
- `/export` 必须包含“MySQL 存储模式”。
- `/stores` 必须包含“门店管理”。
- P0 页面不能显示“第一阶段占位”或“暂未迁移”。

## 后续顺序

1. 继续把 `/inspection`、`/warehouse` 的视觉细节对齐旧 HTML。
2. 给 `/expenses`、`/data-entry`、`/stores` 补正式写入接口和表单。
3. 给 `/export` 补后端生成备份文件、单店利润表、照片 PDF 的下载接口。
4. 给 `/assistant` 接入后端 AI 数据助手接口，问答上下文写 MySQL 审计，不写 localStorage。
