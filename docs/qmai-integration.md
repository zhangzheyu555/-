# 企迈经营数据接入说明

## 本次接入范围

- 使用企迈开放平台固定接口读取授权门店与门店商品营业额。
- 按“门店 × 营业日”异步同步，营业额、成本、退款、商品销量先落 MySQL，再供页面查询。
- 支持月份、门店、营业额/商品销售视图切换和商品关键字筛选。
- 低于 40% 毛利率的门店由后端标记，前端仅负责提示，不在浏览器中重新计算财务口径。
- 支持按当前视图导出 CSV，导出、启动同步、读取授权门店和保存配置均写操作日志。
- 平台配置只保存启用状态、显示名称和门店映射；企迈凭证不进入数据库和浏览器。

## 部署变量

后端启动前配置：

```powershell
$env:QMAI_OPEN_ID='<企迈 openId>'
$env:QMAI_GRANT_CODE='<企迈 grantCode>'
$env:QMAI_OPEN_KEY='<企迈 openKey>'
```

可选变量：

```powershell
$env:QMAI_BASE_URL='https://openapi.qmai.cn'
$env:QMAI_TIMEOUT='20s'
$env:QMAI_MAX_RETRIES='4'
$env:QMAI_CONCURRENCY='4'
```

`QMAI_BASE_URL` 会在后端强制校验为 `https://openapi.qmai.cn`，不能改为任意代理或探测地址。

## 配置和使用顺序

1. 部署人员配置三个凭证环境变量并重启后端。
2. 老板进入“平台配置 → 企迈 → 配置连接”。
3. 读取企迈授权门店，并将每家企迈门店映射到系统内唯一门店。
4. 启用企迈同步并保存。
5. 老板或财务进入“经营财务 → 企迈经营数据”，选择月份后启动同步。
6. 页面轮询批次进度；同步完成后读取已经落库的数据，不直接等待企迈接口返回整月结果。

## 权限边界

- 平台状态：`platform.read`。
- 凭证状态、门店发现和映射配置：`platform.manage`。
- 查询经营数据：`finance.profit.read`，并继续应用财务门店数据范围。
- 启动同步：`finance.profit.write`，且角色必须是老板或财务。
- 导出：`finance.export`，并继续应用财务门店数据范围。
- 未登录由认证层返回 401；无权限或跨门店访问由权限层返回 403。

## 数据表

- `qmai_platform_config`：启用状态和展示名称，不含密钥。
- `qmai_store_mapping`：企迈门店与系统门店的一对一映射。
- `qmai_sync_batch`：异步批次、任务数、失败数和错误摘要。
- `qmai_daily_sales`：门店日营业额、成本和退款。
- `qmai_product_sales`：保留商品 ID、SKU、原始名称和分类的门店日商品销售。

## 明确暂缓

下列同事代码未接入，需单独评估后再决定：

- 企迈控制台 Cookie/Token 通道。
- 可输入任意路径的接口探测能力。
- 自动创建门店账号或固定密码。
- 同事包中的本地启动脚本和明文配置。
- 按商品名去规格、跨 SKU 合并的“批发式归一化”。

暂缓原因是凭证安全、权限审计、生产数据污染或商品口径失真风险；不影响当前开放平台营业额和商品销售主链路。
