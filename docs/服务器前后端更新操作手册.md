# 服务器前后端更新操作手册

**适用部署**：Docker Compose 单机部署；服务器目录为 `/root/store-profit/`。  
**访问地址**：`https://ruguotea.cn/admin/`。  
**更新原则**：前端、后端可独立更新；只重建对应服务，不停止或删除 MySQL、Redis、Postgres 数据卷。

> 本手册只适用于当前 Compose 架构：前端镜像从 `frontend-vue/dist/` 打包，后端镜像从 `backend/target/*.jar` 打包。服务器不需要安装 Node、Maven 或 JDK。

## 0. 更新前检查

1. 在本地完成代码审查和测试；不要将密码、Token、数据库备份上传到仓库。
2. 进入宝塔面板的「终端」，执行：

   ```bash
   cd /root/store-profit
   docker compose ps
   ```

   仅当现有服务状态正常时继续。若已有服务处于 `Restarting`、`Exit` 或 `unhealthy`，先处理现有故障。
3. 本次涉及后端代码、Flyway 迁移或重要业务数据时，先备份数据库：

   ```bash
   cd /root/store-profit
   mkdir -p backup
   docker compose exec -T mysql sh -lc \
     'exec mysqldump --no-tablespaces -uroot -p"$MYSQL_ROOT_PASSWORD" --single-transaction "$MYSQL_DATABASE"' \
     > "backup/mysql_$(date +%Y%m%d_%H%M%S).sql"
   ls -lh backup/
   ```

   此命令只读导出；不要把生成的 SQL 文件上传到公共位置。

## 1. 仅更新后端

**适用范围**：修改 `backend/` 中的 Java 代码、接口、配置文件、Maven 依赖，或新增数据库迁移。

### 步骤 1：本地测试并打包

```bash
cd ~/Downloads/--codex-deepseek-assistant-latest/backend
mvn -q test
mvn -q -DskipTests package
```

确认生成 `target/store-profit-backend-0.1.0-SNAPSHOT.jar` 后执行：

```bash
cd ~/Downloads/--codex-deepseek-assistant-latest
tar -czf /tmp/backend-update.tar.gz backend/target/store-profit-backend-0.1.0-SNAPSHOT.jar
```

### 步骤 2：上传产物

通过宝塔「文件」上传 `/tmp/backend-update.tar.gz` 到服务器 `/root/store-profit/`。

### 步骤 3：服务器替换并重启后端

```bash
cd /root/store-profit
mkdir -p backup/backend
cp backend/target/store-profit-backend-0.1.0-SNAPSHOT.jar \
  "backup/backend/store-profit-backend.$(date +%Y%m%d_%H%M%S).jar"
tar -xzf backend-update.tar.gz
docker compose build backend
docker compose up -d backend
docker compose ps backend
docker compose logs backend --tail=100
```

### 步骤 4：验证

```bash
curl -fsS https://ruguotea.cn/api/health
```

期望返回成功状态（包含 `"status":"UP"`），且 `docker compose ps backend` 显示 `healthy`。随后登录管理端，手动验证本次接口或业务流程。

## 2. 仅更新前端

**适用范围**：修改 `frontend-vue/` 中的页面、样式、路由或前端逻辑。

### 步骤 1：本地构建

```bash
cd ~/Downloads/--codex-deepseek-assistant-latest/frontend-vue
npx vite build --base=/admin/
```

必须保留 `--base=/admin/`，否则线上静态资源会请求错误路径，导致白屏或资源 404。

### 步骤 2：打包并上传

```bash
cd ~/Downloads/--codex-deepseek-assistant-latest
tar -czf /tmp/frontend-update.tar.gz -C frontend-vue dist
```

通过宝塔上传 `/tmp/frontend-update.tar.gz` 到服务器 `/root/store-profit/`。

### 步骤 3：服务器备份旧构建、更新并重启前端

```bash
cd /root/store-profit
mv frontend-vue/dist "frontend-vue/dist.backup.$(date +%Y%m%d_%H%M%S)"
tar -xzf frontend-update.tar.gz -C frontend-vue/
docker compose build frontend
docker compose up -d frontend
docker compose ps frontend
docker compose logs frontend --tail=50
```

该流程保留带时间戳的旧 `dist`，便于回退；不使用 `rm -rf` 删除旧版本。

### 步骤 4：验证

打开 `https://ruguotea.cn/admin/`，使用 `Ctrl + Shift + R` 硬刷新，然后检查：

- 页面不白屏，静态资源没有 404；
- 登录和本次改动功能正常；
- 刷新当前业务页面不会出现 404。

## 3. 前后端和数据库同时更新

按以下顺序执行，避免新版前端调用尚未存在的接口，或新版后端读取尚未创建的字段：

1. 执行「0. 更新前检查」并备份数据库。
2. 若有 Flyway 迁移：确认新增脚本只新增、不修改已执行脚本；版本号必须大于**服务器** `flyway_schema_history` 中的最大版本。
3. 按「1. 仅更新后端」完成后端发布，查看启动日志确认迁移成功。
4. 验证 `/api/health` 与关键接口。
5. 按「2. 仅更新前端」发布前端，硬刷新后完成浏览器验收。

查看服务器已执行的最新迁移版本（仅查询）：

```bash
cd /root/store-profit
docker compose exec -T mysql sh -lc \
  'exec mysql -uroot -p"$MYSQL_ROOT_PASSWORD" "$MYSQL_DATABASE" -e "SELECT version, description, success FROM flyway_schema_history WHERE version IS NOT NULL ORDER BY installed_rank DESC LIMIT 10;"'
```

## 4. 回退

### 前端回退

用本次更新前创建的备份目录恢复：

```bash
cd /root/store-profit
mv frontend-vue/dist "frontend-vue/dist.failed.$(date +%Y%m%d_%H%M%S)"
mv frontend-vue/dist.backup_<更新时间> frontend-vue/dist
docker compose build frontend
docker compose up -d frontend
```

### 后端回退

将更新前自动保存的旧 JAR 放回原文件名后执行：

```bash
cd /root/store-profit
cp backup/backend/store-profit-backend.<更新时间>.jar \
  backend/target/store-profit-backend-0.1.0-SNAPSHOT.jar
docker compose build backend
docker compose up -d backend
docker compose ps backend
```

**注意**：应用回退不会自动回退已执行的 Flyway 数据库迁移。数据库恢复会覆盖更新后的业务数据，只能在严重事故、确认备份时间点和影响范围后进行。

## 5. 禁止操作与排错入口

- 不要在日常更新中运行 `docker compose down -v`；`-v` 会删除数据卷。
- 不要修改服务器 `.env` 中的数据库密码、密钥或 CORS 配置后直接提交到仓库。
- 不要修改已执行的 Flyway 迁移文件；修复必须新增迁移。
- 后端启动失败：`docker compose logs backend --tail=100`。
- 前端异常：`docker compose logs frontend --tail=50`，并检查构建命令是否包含 `/admin/` base。
- 网关或 HTTPS 路由异常：`docker compose logs gateway --tail=100`。
