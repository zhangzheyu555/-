### 数据库注意（踩过的坑）

- **新建库必须指定排序规则**：`CREATE DATABASE xxx CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
  用 MySQL 8 默认的 `utf8mb4_0900_ai_ci` 会在 V40 迁移报「Illegal mix of collations」。
- 企迈相关迁移是 **V63~V65**（重编号两次：V54-56、V61-62 均被本分支其他迁移占用）；
  V63 会先 `drop` 掉本分支 V60 旧版企迈实现建的同名表 `qmai_platform_config` 再建新结构。
- 本分支没有引导账号环境变量机制，账号是从旧库 `store_ops_codex_test.auth_user` 整表复制的；
  企迈凭证配置行也是从旧库 `qmai_platform_config` 复制的（两库表结构一致，可直接 `REPLACE INTO ... SELECT`）。

## 三、企迈接入配置

- 凭证存储：数据库表 `qmai_platform_config`（按租户+品牌一行，brand 默认 `ruguo`=茹菓）。
  网页配置入口：「平台配置」页点企迈卡片弹窗，填 openId / grantCode / openKey / 网关 / 版本 / 授权门店。
  openKey 只存后端，接口只回掩码，绝不下发前端。
- 网关：`https://openapi.qmai.cn`，接口版本 `1.0`。
- **授权门店限定**：`shops` 字段格式 `shopId:门店名,shopId:门店名,...`。
  当前配置了 14 家茹菓门店（285275,287952,287953,287956,287958,287961,287962,287963,
  287966,287971,287987,287988,287989,321047）。配置后系统只统计这些店；
  留空则自动发现账号下全部 37 家门店。增减门店改这个字段即可（系统级配置，所有账号一致）。

## 四、页面功能（平台配置 → 企迈面板，共 3 个标签）

### 1. 企迈营业额
- 按月查询全部授权门店：实收、成本、毛利、毛利率、退款，低毛利行高亮，可排序，CSV 导出。
- 数据口径：企迈 `/v3/dataone/item/store/turnover`，按店按日聚合。整月拉取约 1~3 分钟。

### 2. 企迈商品销售
- 与营业额同一次刷新的数据（`/api/qmai/summary` 的 `items` 字段，不额外增加请求量）。
- 两种视图：全门店汇总（同名商品跨店累加）/ 按门店明细；可按门店筛选。
- 统计口径三挡：饮品（默认，自动剔除费用/小料/零食/占位商品）/ 其他 / 全部；CSV 导出。
- 企迈原始数据一行 = 一个商品在一家店一天的销售；`refundNum` 为负数；`pageSize` 最小 10。

### 3. 物料用量（2026-07-17 新增）
- 用途：输入各产品杯数 → 自动算出每种水果的配方用量（公斤）和**折算采购毛重（斤）**。
- 配方数据：`frontend-vue/src/data/fruitUsage.ts`，由桌面《5月产品用量核算表》的
  「单杯用量」「水果出肉率」工作表生成（生成脚本 `/tmp/gen_fruit_usage.py` 的逻辑已注释在文件头）。
  共 56 个产品（中/大杯拆开，果酱/果汁拆到成分）。改配方 → 改 Excel 后重新生成该文件。
- 换算口径：
  - 果肉类：克重 ÷ 出肉率 = 采购毛重（芒果 53%、橙子 54%、凤梨 59.8%、西瓜 80% 等）；
  - 果汁/果酱：按「酱汁类」毛重系数（雪梨汁 1.265、橙汁 1.562、新鲜芒果汁 849/450 等）；
  - 百香果/耙耙柑/羊角蜜/榴莲/羽衣甘蓝无出肉率数据，按 1:1 计并在备注标注。
- 杯数来源：手动输入，或点「用 X月 企迈销量填充杯数」一键带入全门店饮品销量
  （名称归一匹配；企迈名带「大杯」填大杯配方，否则默认中杯；配方表没有的售卖商品单独列出）。
- 结果支持 CSV 导出（含水果表、其他物料表、杯数明细）。

## 五、后端接口一览（`/api/qmai/*`，全部支持 `?brand=`，缺省 ruguo）

| 接口 | 说明 |
| --- | --- |
| GET `/status` | 配置状态 |
| GET/PUT `/config` | 读取（掩码）/ 保存凭证，需平台管理权限 |
| GET `/summary?month=` | 营业额（shops）+ 商品销量（items），页面主数据源 |
| GET `/probe-shops` | 诊断：发现账号下门店 |
| POST `/probe` | 诊断：探测任意企迈开放平台接口，`{"path":"...","params":{...}}` |
| POST `/console-probe`、GET `/console-income` | 令牌复用通道（商户后台网关），为将来非茹菓品牌预留 |

探测返回码速查：`43004`=路径不存在，`110094`=存在但无权限，`100002`=有权限但参数错，
`1033005`=门店不存在（通常是门店编码问题）。
