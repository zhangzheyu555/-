## 本轮发现并修复

员工打开考试中心概览首次返回 `500`。根因是 `ExamCenterRepository.campaigns` 的动态 SQL 将员工自有任务条件与 `where`、`group by` 直接拼接，产生 `?where` 与 `?)group`。已为动态片段写入明确分隔，并在 `ExamLearningFlowTest` 的 H2 迁移全流程中断言员工自有活动查询。定向考试回归和随后 `mvn -q clean package` 均通过，重启候选后才发布及提交真实 QA 合成考试。

临时 BOSS、督导、员工和店长会话均已登出。为继续 G4 的审计、范围和最终清理验证，暂保留门店 A/B、合成员工、试卷、活动、任务、答卷及其审计；G4 收尾时统一通过候选 API 与基线清单恢复。未执行移动端、WebKit 或 G5 的考试 CSV/视频/附件故障专项测试。
