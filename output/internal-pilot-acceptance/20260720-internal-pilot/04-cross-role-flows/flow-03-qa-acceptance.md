# G4 FLOW-03 真实隔离 QA 验收

执行时间：`2026-07-21`
业务选择：方案（2）。本 FLOW 明确分成两条不可互相替代的链：`区域仓申请 → 总仓审批/发货 → 区域仓收货`，以及 `店长基于已配送单的独立退货 → 总仓审核/收货`。

## 结论

`PASS`。两条链都在专用 loopback QA MySQL 和候选 API 中完成；没有把配送退货当作仓间调拨的反向步骤。

## 隔离与角色范围

| 项目 | 结果 | 证据 |
| --- | --- | --- |
| QA 边界 | PASS | 仅连接 `ai-profit-os-qa-g4-mysql` 的 `127.0.0.1:13307` 和 QA profile 候选服务 `127.0.0.1:18181`。未访问局域网、云端或生产。 |

## A. 仓间调拨闭环

区域仓仓库员以同一创建幂等键重复提交“荆州总仓 → 山东分仓”的 3 单位调拨，返回同一业务单；随后区域仓提交、总仓审批、总仓发货、区域仓收货，并重复收货请求。

| 断言 | 结果 |
| --- | --- |
| 状态顺序 | `DRAFT → SUBMITTED → APPROVED → SHIPPED → RECEIVED` |
| 创建与收货幂等 | 创建重复不新增单据；收货重复返回成功且不重复过账 |
| 路线与金额 | 唯一已收货单为 `总仓(1) → 区域仓(2)`，金额 `30.00` |
| 明细 | 申请/审批/发货/实收均 `3.00`；预留与在途归零；单位成本 `10.00` |
| 总仓库存 | 在手 `22.00`、预留 `0.00`、在途 `0.00`、成本 `10.00` |
| 区域仓库存 | 在手 `3.00`、预留 `0.00`、在途 `0.00`、成本 `10.00` |
| 库存流水 | 总仓预留 `+3.00`、总仓发货 `-3.00`、区域仓在途 `+3.00`、区域仓收货 `+3.00 / 在途 -3.00`，成本均为 `10.00` |
| 调拨动作去重表 | `SUBMIT`、`APPROVE`、`SHIP`、`RECEIVE` 各一条 |
| 操作日志 | 创建、提交、批准、发货、完成收货各一条，全部 `tenant_id=1` |

拒绝边界：区域仓仓库员读取总仓概览为 `403`；匿名读取本调拨为 `401`。

## B. 独立配送退货闭环

店长 A 仅以 FLOW-01 的已收货叫货单为来源，提交 2 单位配送退货和一份纯合成文本附件；总仓审核并确认收货。该退货没有引用或改变上述仓间调拨单。

| 断言 | 结果 |
| --- | --- |
| 状态顺序 | `SUBMITTED → APPROVED → RECEIVED` |
| 来源与收货仓快照 | 来源为门店 A 已配送单；收货仓为荆州总仓 |
| 退货金额与批次 | 一条明细，数量 `2.00`、单价/退货价 `10.00`、金额 `20.00`，保留原出库批次关联 |
| 库存回转 | 总仓批次/在手库存为 `24.00 / 24.00`，成本 `10.00`；门店 A 实存为 `3.00`，退货流水一条、数量 `-2.00`；总仓 `RETURN_IN` 流水一条、数量 `+2.00` |
| 附件 | 退货响应和 SQL 都为 1 个附件；店长回读得到 `200`、`text/plain`，内容 SHA-256 与提交的合成文本一致；下载审计一条且 `tenant_id=1` |
| PDF | 总仓下载退货 PDF 得到非空 `application/pdf`；匿名下载为 `401` |
| 操作日志 | 提交、审核通过、确认收到、下载退货 PDF 各一条，全部 `tenant_id=1` |

## 不在本 FLOW 结论内

- 未执行移动端或 WebKit。
- 未执行 G5 对附件格式/大小/缺失/跨资源攻击、PDF A4 黑白可读性和对象存储故障的专项门禁。
- FLOW-03 通过不代表 G4 通过；FLOW-04 至 FLOW-07 仍需逐项完成。
