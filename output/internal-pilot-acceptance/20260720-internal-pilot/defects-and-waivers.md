- QA：V72→V74 成功；店长真实 API 返回 200；表头 `version=0`、状态 `SUBMITTED`，明细 `shipped_quantity=0.00`，审计 1 条；同幂等键重复提交返回同一 ID，表头/明细/审计/幂等记录均保持 1；测试数据清理后残留 0。

## IP-P1-005 全新 MySQL 8 空库在 V40 排序规则冲突

- 严重度：P1
- 状态：RESOLVED
- 现象：远程 MySQL 8.0.46 空库执行 Flyway 时，V40 报 `Illegal mix of collations` 并中断；已升级的 QA 库不受影响。
- 根因：MySQL 8 默认空库为 `utf8mb4_0900_ai_ci`，历史 V40 临时表未显式声明排序规则，与项目历史表的 `utf8mb4_unicode_ci` 比较失败。
- 修复：空库验证脚本校验数据库名后，在迁移前把受控 CI 数据库统一为 `utf8mb4_unicode_ci`；Flyway 期望版本同步为 V74。未修改既有迁移校验和，也未改真实业务数据。
- 回归：本机 MySQL 8.0.46 默认空库从 `utf8mb4_0900_ai_ci` 受控转换后，74 个迁移成功、最新 V74、应用健康 UP；远程 Release CI 的 MySQL 8/Flyway 任务通过。
- 豁免：无。
