# AI Profit OS Codex Project Rules

本文件是 Codex 在本项目中的工作规则。每次进入本项目时，必须先读取本文件，再进行代码修改、测试、部署和验证。

## 项目名称

AI Profit OS / 多门店经营异常处理系统

## 项目定位

这是一个多门店经营管理系统。核心不是单纯报表，而是把经营异常、今日待办、仓库叫货、采购入库、配送出库、退货单、附件、操作日志、权限角色串成闭环。

## 当前主线

- `frontend-vue` 是唯一正式前端，使用 Vue 3、TypeScript、Pinia 和 Vue Router。
- `backend` 是唯一正式后端，使用 Spring Boot、JdbcTemplate、Flyway 和 MySQL。
- MySQL 是唯一真实业务数据源；浏览器 `localStorage` 只能保存登录会话，不能保存业务数据。
- 老板今日待办只负责提醒、跟踪和跳转；复杂操作必须回到利润、报销、仓库、巡检等业务中心完成。
- 仓库中心覆盖店长叫货、仓库发货、采购入库、库存预警、出入库记录、退货单和附件。
- 所有金额、利润、成本占比等后端计算必须使用 `BigDecimal`。
- 生产环境不得创建固定账号、默认密码、示例业务数据或模拟平台营业数据。

## 目录规则

- `frontend-vue`：唯一正式前端。所有新增页面、业务交互、权限菜单和 UI 改动都在这里实现。
- `backend`：唯一正式后端、接口、数据库迁移和部署产物。
- `backend/src/main/resources/static`：历史静态资源兼容副本，不再作为正式前端入口。
- `runtime-static`：历史运行时静态资源兼容副本，不再继续开发；仅用于旧系统只读回看或紧急回退。
- 顶层 `index.html`、`database.js`：旧版业务与视觉参考，不得作为新功能的数据源或正式入口。
- `web`：实验目录，停止继续开发；不得作为生产入口或新增功能位置。
- 旧 HTML 后续统一通过 `/legacy` 只读入口提供；迁移边界见 `docs/legacy-migration-inventory.md`。
- `backend/src/main/resources/db/migration`：数据库迁移脚本。
- `docs`：设计文档、接口文档、业务说明。
- `output`：验证输出、临时 PDF、截图等结果文件。
- 不要删除用户已有文件。
- 不要回滚用户已有修改。

## 开发规则

- 修改前先读现有代码和迁移脚本。
- 保持最小改动。
- 优先复用现有接口、服务、Repository、响应结构。
- 不要为了新功能破坏已有老板、财务、督导、店长、仓库、运营角色功能。
- 真实业务数据必须落 MySQL。
- 测试数据和真实数据要有区分。
- 页面文案必须中文业务化，不展示英文技术状态码给业务用户。
- 数据库结构变更只能新增 Flyway 迁移，绝不改写已执行迁移。
- 数据库密码、平台密钥和 AI 密钥只能来自环境变量或部署密钥系统，不能写入前端或提交到仓库。
- 所有 Controller 都必须从认证上下文取得当前用户，未经认证返回 401，无权限和跨门店访问返回 403。
- 关键修改、审批、导出、上传下载和权限拒绝必须写操作日志。

## 仓库业务规则

- 店长只能看库存、叫货、查看叫货单、确认收货。
- 仓库管理员可以维护商品、采购入库、处理叫货、发货、设置预警、下载单据。
- 老板可以查看全部风险和单据。
- 财务可以查看采购、入库、单据相关数据。
- 商品停用后不能叫货、不能新增入库，但历史记录和打印单仍可查看。

## 打印单规则

- 入库单、出库单、配送退货单必须从 MySQL 查询生成 PDF。
- 不允许从前端 DOM 临时拼 PDF。
- 下载 PDF 必须写 `operation_log`。
- PDF 要适合 A4 黑白打印。

## 权限规则

- 系统管理员：维护账号、角色、门店数据范围和基础配置。
- 老板：查看全部门店的经营指标、风险和复核事项。
- 财务：处理利润、工资、报销、采购单据和数据核对。
- 店长：仅处理所属门店的数据录入、报销、整改、叫货和收货。
- 督导：处理巡店、问题记录、整改和复核。
- 仓库管理员：处理商品、采购、库存、发货、退货和预警。
- 运营：处理平台配置、培训考试、盘存和数据辅助，不得读取无权限财务数据。
- 前端菜单只显示角色常用功能，但不能替代后端授权检查。
- 未登录必须返回 401；无权限、跨门店和越权附件访问必须返回 403。

## 验证规则

每次完成后至少执行：

- 后端测试与打包。
- Vue3 类型检查与生产构建。
- 在空 MySQL 数据库验证新增 Flyway 迁移。
- 用接口验证未登录、错误角色、跨门店和关键业务流程。
- 如涉及页面，验证 `frontend-vue` 的桌面和移动端视图。
- 明确说明哪些验证通过，哪些因环境或外部依赖未验证。

## 推荐命令

在 `backend` 目录执行：

```powershell
mvn -q test
mvn -q -DskipTests package
$env:MYSQL_PASSWORD='<本机MySQL密码>'; java -jar target/store-profit-backend-0.1.0-SNAPSHOT.jar
```

Linux/macOS 环境可用：

```bash
MYSQL_PASSWORD=<本机MySQL密码> java -jar target/store-profit-backend-0.1.0-SNAPSHOT.jar
```
